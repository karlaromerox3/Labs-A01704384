• info¿Por qué es una buena práctica separar el controlador de la vista?
    Porque permite separar los sistemas en información y lógica, e interfaz de usuario. De esta manera, se habla de un controlador que se encarga de las entradas, procesamiento, salida y busqueda de datos; y otro para entregar los dat.os en una presentación amigable para el usuario. Toda esta división permite mayor mantenibilidad y modularidad.
    Referencia: https://codingornot.com/mvc-modelo-vista-controlador-que-es-y-para-que-sirve
  • infoAparte de los arreglos $_POST y $_GET, ¿qué otros arreglos están predefinidos en php y cuál es su función?
    $_SERVER Contiene información de cabeceras, rutas y ubicaciones de script del servidor web.
    $_FILES Asociación de elementos que se han subido usando POST.
    $_COOKIE Asociativo de variables pasadas al script con Cookies.
    $_SESSION Tiene un conjunto de variables de sesión para el script que se este corriendo.
    $_REQUEST Tiene lo mismo que contiene $_GET, $_POST y $_COOKIE.

    Referencia: https://www.php.net/manual/es/language.variables.superglobals.php

  • info¿Qué es XSS y cómo se puede prevenir?
    Es un tipo de ataque de código malicioso que se envía a través de scripts a un usuario. Normalmente son causados porque no se validan los campos de input en las páginas web.
    Se pueden prevenir usando htmlentities y htmlspecialchars.
    Referencia: https://owasp.org/www-community/attacks/xss/
