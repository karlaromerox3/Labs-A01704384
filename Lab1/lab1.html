Conceptos Importantes

URL

Significa Uniform Resurce Locator y es el equivalente a las direcciones IP, que son un ubicador para cualquier equipo que se conecte a una red. Dado que las IP son un dato numero muy largo, se crearon las URL que son como un apodo más amigable para que el usuario ubique mejor los recursos en la web. [1]

MIME

Sus siglas significan Multipurpose Internet Mail Extension y es el formato creado por IETF para los adjuntos de correo que no son texto; es decir, permite que uno mande y reciba audio, fotos, videos, etc.[2]

HTTP

Este protocolo tiene una serie de códigos de estado para las peticiones y mensajes que se mandan entre servidores y clientes. Algunos de los más importantes son:
1XX: Respuesta informativa (normalmente de que la petición esta siendo procesada)
2XX: Indican éxito en la petición y que el procesamiento fue correcto.
3XX: Los mensajes de redirección quieren decir que el usuario tiene que hacer algo extra para que el request sea terminado.
4XX: Significan errores en el cliente aunque normalmente el servidor da información del porqué,normalmente es porque la petición es erronea.
5XX: Se refieren a un error en el que el servidor no puede procesar la petición y es error del administrador del servidor.[3]

HTML5

Desde el 2000, el W3C puso a HTML como el lenguaje oficial del Internet, y poco a poco ha tenido gran auge pues le da un aspecto al código mucho más estructurado y organizado, separando totalmente la estructura de los estilos. Algunas de las características principales son que redujo la dependencia de extensiónes, tiene mejor manejo e inserción de multimedia y visualización sin tener que estar conectado a internet.[4]

JS

JavaScript es un lenguaje que se usa para controlar sitios web con efectos dinamicos, agregar interactividad a las páginas web o crear juegos en buscadores web. Se usa para automatizar y animar páginas web y por eso se le denomina un "scripting language"[5]

PHP

Es el acronimo para Hypertext Pre-Processor y entre muchas cosas, permite el scrpting del lado del servidor, hacer aplicaciones de escritorio y aplicaciones web con un gran soporte a bases de datos. PHP se ejecuta en el servidor y genera código de HTML que después se envía al cliente, para que vea solo lo que el desarrollador quiere que sea visible.[6]

JSP

Es una tecnología para poder integrar código de Java en una página web. Esto se logra con código en HTML y XML y se basa en los clasicos servlet de Java. Normalmente se identifican por estar entre un bracket y un signo de porcentaje. [7]

Ajax

Es una tecnología que combina varias técnicas de JavaScript y el uso de XMLHTTPREQUEST y, a grandes rasgos, permite intercambiar datos con el servidor y actualizar partes de un sitio web sin necesidad de cargar completamente la página por medio de callbacks y llamadas asíncronas. Ajax se usa en el rating de películas en Netflix, pues el usuario califica algúna serie y este se guarda en la base de datos si tener que volver a cargar toda la página.

Preguntas

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet se refiere a una red de redes, que está distribuida por todo el mundo. Es la interconexión de redes de computadoras, servidores y dispositivos que tienen posibilidad de ser parte de una red de computación. Por otro lado, la World Wide Web es un sistema para acceder a dispositivos conectados en Internet. Se puede acceder a ella por medio de buscadores de internet y utiliza HTTP para acceder y mostrar distinta información. De esta manera puede decirse que el Internet es el hardware de miles de dispositivos conectados y la WWW es uno de los sistemas que nos permiten acceder al contenido que ofrece el internet. [1]

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
  • GET: Pide recursos desde un servidor web, segun una condicion o rango determinado, la petición viaja en el encabezado del mensaje y sin cifrar.
  • HEAD: Recupera información sobre algun recurso, pero no el recurso en sí. Es mucho más rápido que el GET ya que pasa menos información y es usado para ver cuando se modificó por última vez y determinar si es factible usar una copia local o no. [2]
  • POST: Envía datos al servicio web, como recursos o información de un nuevo registro en una base de datos. La información viaja en el cuerpo del mensaje, por lo que es más segura.
  • PUT: Reemplaza recursos en el servidor, si no existe previamente, funciona como un POST.
  • PATCH: Permite actualizar o hacer cambios en alguna parte del recurso pero no en todo, esto con la finalidad de no tener que subir todo el archivo y usar un exceso de banda ancha. [3]
  • DELETE: Elimina recursos o registros que están en el servidor. [4]

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Los errores que empiezan con 2XX son referentes al éxito del método enviado. Más especificamente una respuesta con el código 200 significa que todo fue como se esperaba, la comunicación y el procesamiento fueron exitosos. [5]

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Los códigos 4XX significan errores del cliente en la solicitud al servidor. Un error 404 quiere decir que no se encontró el recurso que se solicitaba y que el servidor no sabe si alguna vez existió. La responsabilidad de repararlo es del desarrollador, pues quiere decir que los GET no están bien creados o el enlace está mal.[5]

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Los códigos que tienen formato 5XX hablan de errores en el servidor, quien pudo aceptar la solicitud pero por algún error interno no pudo contestar. El estado 500 quiere decir "error interno en el servidor". Este caso no es responsabilidad del desarrollador, si no del administrador del servidor ya que puso un programa que estaba mal.[5]
