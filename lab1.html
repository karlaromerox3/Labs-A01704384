¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet se refiere a una red de redes, que está distribuida por todo el mundo. Es la interconexión de redes de computadoras, servidores y dispositivos que tienen posibilidad de ser parte de una red de computación. Por otro lado, la World Wide Web es un sistema para acceder a dispositivos conectados en Internet. Se puede acceder a ella por medio de buscadores de internet y utiliza HTTP para acceder y mostrar distinta información. De esta manera puede decirse que el Internet es el hardware de miles de dispositivos conectados y la WWW es uno de los sistemas que nos permiten acceder al contenido que ofrece el internet. [1]

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
  • GET: Pide recursos desde un servidor web, segun una condicion o rango determinado, la petición viaja en el encabezado del mensaje y sin cifrar.
  • HEAD: Recupera información sobre algun recurso, pero no el recurso en sí. Es mucho más rápido que el GET ya que pasa menos información y es usado para ver cuando se modificó por última vez y determinar si es factible usar una copia local o no. [2]
  • POST: Envía datos al servicio web, como recursos o información de un nuevo registro en una base de datos. La información viaja en el cuerpo del mensaje, por lo que es más segura.
  • PUT: Reemplaza recursos en el servidor, si no existe previamente, funciona como un POST.
  • PATCH: Permite actualizar o hacer cambios en alguna parte del recurso pero no en todo, esto con la finalidad de no tener que subir todo el archivo y usar un exceso de banda ancha. [3]
  • DELETE: Elimina recursos o registros que están en el servidor. [4]

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Los errores que empiezan con 2XX son referentes al éxito del método enviado. Más especificamente una respuesta con el código 200 significa que todo fue como se esperaba, la comunicación y el procesamiento fueron exitosos. [5]

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Los códigos 4XX significan errores del cliente en la solicitud al servidor. Un error 404 quiere decir que no se encontró el recurso que se solicitaba y que el servidor no sabe si alguna vez existió. La responsabilidad de repararlo es del desarrollador, pues quiere decir que los GET no están bien creados o el enlace está mal.[5]

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Los códigos que tienen formato 5XX hablan de errores en el servidor, quien pudo aceptar la solicitud pero por algún error interno no pudo contestar. El estado 500 quiere decir "error interno en el servidor". Este caso no es responsabilidad del desarrollador, si no del administrador del servidor ya que puso un programa que estaba mal.[5]

Que la W3C no se recomienda su uso en HTML5 porque hay alternativas mucho más flexibles y fáciles de usar con hojas de estilo de CSS. Algunos de los tags depreciados son los de estilo, como font, center, u,s, entre otros y algunos atributos son: align, background, bgcolor, border, link, etc.[6]

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

En términos generales, HTML5 simplfica muchos tags, tiene mejor manejo de errores, soporte para multimedia, tags para canvas,video y audio, separa el estilo del contenido, además de que soporta drag and drop de archivos sin necesidad de ninguna extensión. [7]
